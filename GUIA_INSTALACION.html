🚀 Opción A — Uso Inmediato (Sin Instalación)
✅
Esta es la forma más rápida. El MVP funciona directamente en el navegador, sin necesidad de instalar nada. Los datos se guardan localmente en el navegador.
1
Abre el archivo principal

Haz doble clic en app/index.html para abrir el sistema en tu navegador.

2
Navega a los módulos avanzados

Desde el menú lateral, accede a Agenda Visual, Dashboard Financiero, Aseguradoras y Documentos Clínicos.

3
Comienza a registrar pacientes

Ve a Pacientes → Nuevo Paciente y completa el expediente.

⚠️
Limitación: Los datos se guardan en el navegador (localStorage). No se comparten entre computadoras. Para uso en red, usa la Opción B o C.
🐍 Opción B — Backend Python Local (Red Local)

Para usar el sistema en red local (múltiples computadoras) con base de datos centralizada.

Requisitos previos

SoftwareVersiónDescarga
Python3.10+python.org
PostgreSQL15 o 16postgresql.org
Git (opcional)Cualquieragit-scm.com
1
Crear la base de datos PostgreSQL
psql -U postgres
CREATE DATABASE consultorio_db;
CREATE USER medico WITH PASSWORD 'tu_password_segura';
GRANT ALL PRIVILEGES ON DATABASE consultorio_db TO medico;
\q
2
Ejecutar el esquema SQL
psql -U medico -d consultorio_db -f database/schema.sql
3
Configurar el archivo .env
copy .env.example .env
# Editar .env con tus datos

O ejecutar el asistente de configuración:

python setup.py
4
Instalar dependencias e iniciar
cd backend
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
5
Acceder al sistema

API Docs: http://localhost:8000/docs

Frontend: Abrir app/index.html en el navegador

La opción más robusta: incluye base de datos, API, frontend y backups automáticos.

# Editar .env con contraseñas seguras

Levantar todos los servicios

Acceder

Comandos útiles

docker-compose ps

# Ver logs en tiempo real
docker-compose logs -f backend

docker-compose down

# Backup manual de la BD

docker-compose --profile admin up -d

🎯 Primeros Pasos tras la Instalación
1
Configurar datos del médico

En el archivo .env, actualiza MEDICO_NOMBRE, MEDICO_ESPECIALIDAD, MEDICO_CEDULA y MEDICO_TEL. Estos datos aparecen en todos los documentos impresos.

2
Registrar primeros pacientes

Ve a Pacientes → Nuevo Paciente y crea los expedientes. Puedes importar datos usando el módulo de reportes (CSV).

3
Configurar aseguradoras

En el módulo Aseguradoras ↗, edita los datos de contacto, teléfonos de autorización y procedimientos cubiertos de cada aseguradora.

4
Personalizar plantillas de documentos

En Documentos Clínicos ↗, verifica que las plantillas quirúrgicas reflejen tu práctica clínica. Puedes editar los textos directamente en el generador.

5
Programar citas

En Agenda Visual ↗, crea las citas del día. Las vistas mensual, semanal y diaria te permiten visualizar la agenda completa.

🛟 Solución de Problemas Comunes

El archivo HTML no abre correctamente

Asegúrate de abrir los archivos con un navegador moderno (Chrome, Edge o Firefox). Haz clic derecho → "Abrir con" → tu navegador.

Los datos no se guardan entre sesiones

El MVP usa localStorage del navegador. No borres el caché del navegador si quieres conservar los datos. Para datos persistentes usa la Opción B o C.

Error de conexión a PostgreSQL

Verifica que PostgreSQL esté corriendo: pg_isready. Confirma que el usuario y contraseña en .env sean correctos.

Error al instalar WeasyPrint

WeasyPrint requiere Cairo y Pango. En Windows, descarga las dependencias desde el sitio oficial de WeasyPrint o usa el Dockerfile que ya incluye todo.

Docker no inicia

Asegúrate de que Docker Desktop esté abierto y corriendo antes de ejecutar docker-compose up.
